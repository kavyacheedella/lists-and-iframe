WEBSITE

  • A website is a collection of web pages and related content that is identified by a common domain name and published on at least one web server. It is typically accessible via the Internet or a private local area network (LAN) using a web browser. Websites can serve various purposes, such as providing information, entertainment, commerce, communication, or interaction with users. They may contain text, images, videos, hyperlinks, forms, and other multimedia elements to convey information or facilitate user interaction. Websites can range from simple static pages to complex dynamic web applications with interactive features and databases.

WEB APPLICATION

  • A web application is a software program that users interact with through a web browser over the internet. It's different from a static website because it can perform various functions like processing data, storing information, or allowing users to interact with it in different ways, such as filling out forms, uploading files, or conducting transactions. Examples of web applications include email services like Gmail, social media platforms like Facebook, and online shopping websites like Amazon.

LIBRARY

  • Library is a collection of pre-written code modules, functions, or classes that developers can use to perform specific tasks or solve common problems. These libraries are designed to be reusable and often focus on providing functionality for specific areas of development.

FRAMEWORK

  • In web development a framework is a pre-established collection of libraries, tools, and conventions that provide a structured foundation for building web applications.

HTML - HyperText Markup Language

  1. HTML is a Markup Language

    • Hypertext: "Hypertext" refers to text displayed on a computer or other electronic device with references (hyperlinks) to other text that the reader can immediately access. These hyperlinks can lead to different sections of the same document, other documents, or entirely different websites. The concept of hypertext is central to HTML, as it allows for the creation of interconnected documents and is fundamental to the structure of the web..

    • Markup: A set of rules that defines how the layout and content should be displayed in the browser. It focuses on the presentation and structure of content.

    HTML History


    1. Initial Version (1991)

      • Created by Tim Berners-Lee.

      • Consisted of 18 elements.

    2. HTML 2.0 (1995)

      • First standardized version.

      • Introduced forms and basic multimedia support.

    3. HTML 3.0 (1997)

      • Introduced tables, applets, and more complex layout features.

      • Never fully adopted.

    4. HTML 4.0 (1997)

      • Introduced support for scripting languages like JavaScript.

      • Added support for CSS (Cascading Style Sheets).

    5. HTML 5.0 (2014)

      • The latest major version.

      • Introduced new elements, attributes, and behaviors.

      • Improved support for multimedia and graphical content.

    Summary


    • HTML is the foundational language for creating web pages.

    • It uses tags and elements to define the structure and content

    • The language has evolved significantly since its inception, with HTML5 being the current standard, offering enhanced capabilities for modern web development.

    • Elements that start on a new line and take up the full width available.

    • Elements that do not start on a new line and only take up as much width as necessary.

    Hr tag

    The hr tag in HTML is used to create a horizontal rule, also known as a horizontal line or divider, within the content of an HTML document. It is a self-closing tag. When the browser renders the HTML, it displays a horizontal line across the width of its containing element, typically to visually separate sections of content.

    Pre tag

    The pre tag in HTML stands for "preformatted text". It is used to display text exactly as it appears in the HTML code, preserving both spaces and line breaks. This tag is particularly useful for displaying code snippets, ASCII art, or any text where maintaining the original formatting is important.

    Path:

    path specifies the location of a resource on the web server. It follows the domain name and precedes any query parameters or fragment identifiers.

    https://www.example.com/folder/subfolder/file.html

    In this example, the path is "/folder/subfolder/file.html", which indicates the location of the file "file.html" within the "subfolder" of the "folder" directory on the server.

    Absolute Paths

    An absolute path is a complete URL that includes the protocol (e.g., http:// or https://), domain name, and the full path to the resource. It provides the exact location of a resource on the web.

    Example: https://www.example.com/folder/subfolder/file.html

    Advantages of Absolute Paths:

    1. Consistency: Always points to the same resource, regardless of the current page's location.

    2. External Resources: Useful for linking to external resources or when the resource is hosted on a different domain.

    Disadvantages of Absolute Paths:

    1. Maintenance: If the domain or structure changes, all absolute links need to be updated.

    2. Long URLs: Can result in longer URLs, making them less readable.

    Relative Paths

    A relative path is a URL that specifies the location of a resource in relation to the current page's location. It does not include the domain name or protocol, making it shorter and more flexible.

    Example: subfolder/file.html

    Advantages of Relative Paths:

    1. Flexibility: Automatically adjusts if the current page is moved to a different directory.

    2. Shorter URLs: More concise and easier to read.

    Disadvantages of Relative Paths:

    1. Context-Dependent: Depends on the current page's location, which can lead to broken links if the structure changes.

    2. Limited to Same Domain: Cannot be used to link to resources on different domains.

    Anchor tags

    • Anchor tags in HTML are used to create hyperlinks that allow users to navigate from one page to another or to a specific section within the same page. The anchor tag is represented by the <a> element, and it typically includes an href attribute that specifies the URL or target of the link.

    Example:

    Visit Example Website

    In this example, clicking on the link will take the user to "https://www.example.com".

    Anchor tags can also be used to link to specific sections within the same page by using fragment identifiers. A fragment identifier is a part of the URL that starts with a hash symbol (#) and refers to an element with a matching id attribute on the page.

    Example:

    Go to Section 1

    In this example, clicking the link will scroll the page to the element with the id "section1".
